# 협업 캐릭터 모션 (회의실 집결) — 설계 스펙

- 날짜: 2026-07-23
- 대상 파일: `monitor.html` (단일 파일 SPA)
- 상태: 승인됨 (구현 대기)

## 목표

대시보드의 가상 오피스 씬에서 에이전트 캐릭터가 **협업할 때 회의실로 걸어가 함께 회의**하는
모션을 추가한다. 협업이 끝나면 각자 책상으로 돌아온다.

## 협업의 정의 (트리거)

`activity.jsonl` 데이터에는 "A와 B가 대화" 같은 명시적 협업 신호가 없다. 실질적 협업 신호는
**부모 세션이 활성 서브에이전트를 가질 때**뿐이다.

- 트리거: 한 부모 에이전트의 `subs` 중 상태가 `working` 또는 `thinking`인 것이 1개 이상.
- 해제: 해당 부모의 활성 서브에이전트가 모두 종료(`SubagentStop`)되거나 `idle`이 됨.
- 여러 부모가 동시에 조건을 충족하면 회의실 좌석을 나눠 착석한다.

## 이동 단위 (확정: 전자안)

부모 아바타는 **자기 서브에이전트들을 현재처럼 nested 상태로 데리고** 클러스터째 이동한다.
팀이 함께 회의에 들어가는 그림. 서브에이전트를 독립 캐릭터로 승격해 개별 착석시키는 안은
리팩터 규모가 커서 이번 범위에서 제외한다(YAGNI).

## 아키텍처

### 1. 지속형(재조정) 렌더러

현재 `render()`는 매 폴링마다 `#floor`의 `innerHTML`을 통째로 교체하여 애니메이션 상태를
파괴한다. 이를 **id 기준 재조정(diff) 렌더**로 교체한다.

- 아바타 DOM을 `Map<agentId, HTMLElement>`로 유지한다.
- 폴링마다: 새로 등장한 세션만 노드 생성, 사라진 세션만 노드 제거(퇴장 후 정리),
  나머지는 상태 클래스와 문구(task/nameplate)만 in-place 갱신한다.
- 각 아바타는 `position:absolute` + `transform: translate(x,y)`로 배치한다.
- **위치 이동은 CSS transition이 트윈한다.** 별도 rAF 루프 없이 transform transition
  (약 0.8~1.2s ease)으로 "걸어가는" 활공을 만든다.
- `#floor`를 flow 레이아웃에서 좌표 배치로 전환한다.

카드(`#cards`), KPI(`#kpis`), 피드(`#feed`)는 기존처럼 innerHTML 교체 방식을 유지한다
(애니메이션 대상이 아님). 재조정은 `#floor`에만 적용한다.

### 2. 좌표 계층

- **홈(책상) 슬롯**: 바닥(무대 구역)을 따라 에이전트 수에 맞춰 균등 분배한 x 좌표.
- **회의 좌석**: 회의실 테이블 둘레를 원형으로 분할한 좌석 좌표.
- 좌표는 `#scene`/`#floor`의 현재 픽셀 폭 기준으로 계산하고, `resize` 시 재계산한다.
- 반응형: 좁은 폭에서는 회의실 구역이 무대 아래로 접히도록 구역 원점을 폭 기준 분기한다.

### 3. 회의실 구역 (씬 안 신설)

- `#scene` 오른쪽(넓은 폭) 또는 하단(좁은 폭)에 회의실 존을 만든다: 러그 + 원형 테이블 SVG,
  은은한 조명. 무대(책상 구역)와 시각적으로 분리한다.
- 테이블 중앙에 공용 아이콘(공유 문서/전구)을 은은하게 둔다.

### 4. 상태 머신 (에이전트별)

```
at-desk ──(부모에 활성 서브에이전트 발생)──▶ walking-to-meeting ──▶ in-meeting
   ▲                                                                    │
   └──────────── walking-to-desk ◀──(서브에이전트 종료/idle)────────────┘
```

- `deriveView`/`buildAgent`의 데이터 파생 로직은 유지하되, 에이전트에 `meeting`(boolean)
  플래그만 추가로 계산한다(위 트리거 규칙).
- 렌더러는 각 아바타의 목표 위치를 `meeting ? 회의좌석 : 홈슬롯`으로 정하고 transform을
  갱신한다. 이동 중에는 `.walking` 클래스를 부여한다(도착 시 제거).

### 5. 모션 디테일 (전부 transform/opacity)

- **걷기**: transform transition으로 활공 + `.walking` 클래스에 미세 상하 bob 키프레임.
- **회의 중**: 좌석에서 미세 bob, 기존 `.thought` 말풍선을 순번 점등하여 "대화" 느낌.
- 레이아웃 바인딩 속성(width/height/top/left/margin 등)은 애니메이트하지 않는다.

### 6. 접근성

- `prefers-reduced-motion: reduce`: 이동 transition과 bob/말풍선 키프레임을 끄고 위치만
  즉시 스냅한다.
- 아바타의 기존 `role="img"`/`aria-label`은 유지한다.

## 건드리지 않는 것

- 데이터 파이프라인(`deriveView`, `buildAgent`), 카드/KPI/피드 렌더, 승인 API
  (`/api/pending`, `/api/decision`), 폴링 루프, 서버(`approve-server.py`), Docker 구성.

## 범위 / 리스크

- 실질 작업은 `monitor.html`의 RENDER 섹션을 innerHTML 교체 → 재조정 방식으로 바꾸는 것과
  좌표/상태 계층 추가. 약 150~200줄 규모, 단일 파일 내 변경. 데이터·서버 무변경.
- 주요 리스크: `#floor` 좌표 전환에 따른 반응형 레이아웃 회귀. 320/768/1024/1440 폭에서
  겹침·오버플로 확인 필요.

## 검증

- 폭 320/768/1024/1440에서 무대·회의실 배치가 겹치지 않고 오버플로 없는지 스크린샷 확인.
- 서브에이전트 발생/종료를 시뮬레이트한 `activity.jsonl`로 walk→meeting→walk-back 전이 확인.
- `prefers-reduced-motion`에서 스냅 동작 확인.
- 폴링 중에도 아바타가 리셋되지 않고 위치/상태가 이어지는지 확인.
